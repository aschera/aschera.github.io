Python

The creator of Python is 'a big fan of Monty Python's Flying Circus.'

An interpreted language.

Python example Python is a very general purpose language and can be used for almost anything. I found this fun short movie that explains it quite well: What is Python used for (Love, 2014). An example of where Python might be fast and easy to use is to count or add something. For example Python has a build in counter class that is fairly easy to use to count most common words for example (Python, n.d.):

Compiled
Basics compiled code has to be be translated into binary/assembly language before the program is executed (stakx, 2010). 		Interpreted
Python does not need compilation to binary because you can run the program directly from the source code/from the command line (Features, n.d.).
Compiler
This requires a lot of expertise and you have to make sure the proper libraries are linked and loaded etc. for the program to run without error (Features, n.d.). To do this you have to use a compiler, which is a special program you need to download and install first and then run your program through.		 Virtual machine/ Interpreter
Python does have a compiler, but it runs automatically. The Python compiler is a virtual machine (which itself is a compiled program, for example: CPython) that converts the code into an intermediate form called bytecodes. (Understanding, 2014). Running the program afterwards is done through an interpreter which takes the bytecodes and translates them into the native language of your computer, which then can run the program (DeSimone, 2012). The Python interpreter is installed as a program when you first install Python on it./td>
Initial waiting time
Compiling can take anywhere from a few seconds to hours of waiting before the program is ‚Äòdone‚Äô. After the code is compiled, it needs to be tested. This can also take some effort and time.		 No initial waiting time
There is no delay when using Python and it is perfect for being used on demand. Testing python is done while you write it and errors can often be dealt with ‚Äòon the go‚Äô.
High effort when porting
All this makes BASIC code very specific to one computer or system, since implementation of the same code into other systems often requires you to make a lot changes to the source code or recompiling it. Also paths and libraries might be moved or not available and functions might differ in behavior on the new system (stakx, 2010).		 Easily Portable

Fast execution speed
The finished programs are executed directly by the CPU, which makes them startup fast and react to commands almost instantaneous (Grant, 2014). This often means that compiled languages are faster than interpreted, but speed varies with each program/system.		 Slow execution speed
Because of the way Python is first compiled by a virtual machine and then interpreted by another program before it actually executes the program, it is often times slower than a compiled program that can execute without delay.
